浅析复杂度

发表于 2020-05-14 | 分类于 Data Structure
字数统计: 1.3k | 阅读时长 ≈ 5

我们学习数据结构， 很多时候都会提到算法，然而提到算法，算法分好坏，就会说到复杂度，那今天就来简单的看一看这个复杂度。

时间复杂度

我们先通过一道经典的算法题展开今天的正题

求第n个斐波那契数

0, 1, 1, 2, 3, 5, 8, 13……..

上面写出了求斐波那契数两种算法，当对两种算法进行运行时， 传入的n越大， fib1方法的计算速度就越慢，因此，可以得到一个结论，解决同一个问题，使用不同算法，效率可能相差非常大

如何来评判一个算法的好坏

一般从以下维度来评估算法的优劣

正确性，可读性，健壮性(对不合理输入的反应能力和处理能力)

时间复杂度：估算程序指令的执行次数(执行时间)

空间复杂度：估算所需占用的存储空间

在本篇文章中，仅仅只是对算法的时间复杂度进行研究，那如何来计算时间复杂度呢？

大O表示法(Big O)

一般用大O表示法来描述复杂度， 它表示的是数据规模n对应的复杂度

规律

忽略常数，系数，低阶

  • 9 >> O(1)

  • 2n + 3 >> O(n)

  • n^2 + 2n + 6 >> O(n^2)

  • 4n^3 + 3n^2 + 22n + 100 >> O(n^3)

对数阶一般省略底数

  • log2(n) = log2(9) * log9(n)

    所以 log2(n), log9(n)统称为logn

注意： 大O表示法仅仅是一种粗略的分析模型，是一种估算，能帮助我们短时间内了解一个算法的执行效率

在了解以上规律之后， 下面来进行一些练习
